#### 系统架构设计

##### 整体架构分析 

（架构优化思想：筛选有效流量 异步处理数据）

+ 核心功能整体架构
  + 用户-> Web服务器 -> 数据库
    + Web服务器压力大，安全验证成本高
    + 实时读取数据对数据库会造成压力
    + 在高并发下无法保障数据一致性

##### 系统需求分析 

+ 前端页面需承载大流量 
  + CDN 静态回源 
  + SLB 流量负载均衡 分布式安全验证（流量拦截系统 ）-> 秒杀数量控制（防止超卖 增加性能） ->Web服务 （GO服务器集群） RabbitMQ 队列消息服务 -> (防止爆库)Mysql
+ 在大并发状态下解决超卖问题
+ 后端接口需要满足横向扩展
